Methodology

No tricks. No insider access. Just the numbers.

Five things you should know before you trust any benchmark page.

01Same prompts, fixed rotation

Five prompts rotate one per day: waitlist, marketplace, dashboard, blog, CRUD. The full list lives in our public repo — benchmark-prompts.json — never edited mid-cycle without a changelog entry.

02We pay both bills

Both products run on Pro accounts purchased with VibeSDK's own credit card. No insider tier, no comped credits, no special access on either side.

03Wall-clock = preview-ready

Timer starts on POST submit. Stops when the product emits a working preview URL we can open in a fresh browser. Not "first token", not "plan generated" — actually usable.

04Cost normalised to USD

Each product's credit currency converted using their latest published pricing. We re-validate the conversion table weekly. Raw credits also shown in the table so you can do your own math.

05Failures are recorded

If Emergent times out, we record the failure rather than retrying until it works. Same rule applies to us — if our cron fails, the row shows FAIL until we ship a fix.

06Open-source the script

The cron runner is in our public repo: worker/cron/benchmarkRunner.ts. Fork it, run it yourself, prove us wrong. Pull requests welcome.
