### S15 Teams architecture note
vibesdk's DO-per-session model creates a constraint: each `SimpleCodeGeneratorAgent` instance is single-tenant. Multiplayer requires either:
- **Option A:** Multiple WebSocket connections to same DO (already supported via `getWebSockets()` — partial parity possible with permissions layer)
- **Option B:** Team workspace DO that orchestrates per-session DOs (new layer)
- **Decision:** Option A for MVP (share session link → multiple WS viewers → contributor permissions S15 design spike needed)
