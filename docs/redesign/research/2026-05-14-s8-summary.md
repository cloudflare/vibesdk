## Key Design Decisions

| Decision | Rationale |
|---|---|
| Mastra in-memory mode | CF Workers have no persistent Mastra storage. DO SQLite owns all state. |
| runImpl closure pattern | Keeps DO `this` context accessible without Mastra capturing it. |
| No double-eval | Mastra path: workflow runs eval internally, skips runPhaseEvalGate. |
| STATE_DELTA for eval scores | AG-UI standard. RFC 6902 patches at `/phases/<name>/eval*` |
| storePhaseEvalMemory fire-and-forget | Memory outage must never interrupt generation pipeline. |
| NullMemoryClient | Testing + rollback safety — swap without code change. |
| Barrel export (memory/index.ts) | Clean DI: callers depend on interface, not concrete paths. |
