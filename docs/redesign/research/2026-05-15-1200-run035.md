## 3. D1 Auto-Provisioning — Architecture Decision

### Confirmed REST API

Cloudflare REST API confirmed via live docs:
```
POST /client/v4/accounts/{account_id}/d1/database
Headers: Authorization: Bearer {CF_API_TOKEN}
Body: { "name": string, "primary_location_hint"?: "wnam|enam|weur|eeur|apac|oc" }
Response: { result: { uuid, name, created_at, jurisdiction, ... }, success: true }
```

### Design for Sprint 3 P0

**Scope:** When a user's generated app needs a database:
1. `worker/services/d1/D1ProvisionService.ts` — creates D1 via CF REST API using
   `env.CLOUDFLARE_API_TOKEN` + `env.CLOUDFLARE_ACCOUNT_ID` (already in worker env)
2. Returns `{ databaseId, databaseName }` to the calling context
3. The blueprint LLM receives D1 binding scaffolding instructions (useCase
   instructions already include D1 schema guidance for AI SaaS + payments)

**What vibesdk does NOT do (Sprint 3):**
- Automatically add D1 binding to the user's generated app's wrangler.jsonc
  (requires writing to the user's CF account, which needs user API token)
- Run migrations on behalf of user (too much infra control)

**Realistic Sprint 3 scope:** Provide the user with:
a. The D1 database UUID (created in vibesdk's own CF account, used for preview)
b. A generated `wrangler.setup.md` with exact commands to provision in their
   own CF account

**Decision DEC-035-D:** D1 auto-provisioning Sprint 3 = vibesdk-owned preview
D1 (per-session, automatically provisioned on first DB request) + generated
`wrangler.setup.md` for user's own CF account deployment. `D1ProvisionService`
implementation: `createSessionDatabase(sessionId)` → CF REST API → returns
uuid. Session cleanup: delete via `DELETE /client/v4/accounts/{id}/d1/database/{uuid}`
on session expiry.

## 5. Lovable May 2026 — New Signals

### Wiz Scanning Native (confirmed May 2026)

Wiz security scanning is now natively integrated (not just marketed) — findings
surface in every Lovable project's Security view AND flow into Wiz Code and
Build scans. This is a deeper integration than run033 indicated.

**vibesdk counter:** IsolationBadge + SECURITY.md "Isolated by Architecture"
narrative remains the correct counter. Wiz = active scanning of generated code
for CVEs. vibesdk = structural isolation prevents entire classes of BOLA/injection
attacks before they occur. Different paradigms. Our narrative: "Scan code when
you don't control the infrastructure. We control the infrastructure."

### Lovable SEO + AI Search (May 13, 2026 — NEW GAP)

Lovable now generates SEO meta tags, Open Graph images, and AI-search
(llms.txt) compatibility for deployed apps. This means Lovable-generated apps
are indexable and AI-search-friendly out of the box.

vibesdk currently does NOT generate SEO meta tags or llms.txt for generated apps.

**Decision DEC-035-F:** SEO scaffolding (meta tags + OG image placeholder +
llms.txt) — add to `PLAUSIBLE_ANALYTICS_HINT` companion or as a separate
`SEO_SCAFFOLDING_HINT` in getUsecaseSpecificInstructions. Sprint 3 P1.
