## 1. Lovable: Payments + Desktop + Connector Expansion (April 24, 2026)

**Source:** [Lovable changelog](https://docs.lovable.dev/changelog)

The most recent changelog entries (April 24) reveal features not captured in run016 (which covered the May 7 MCP server preview):

### Built-in Payments (Paddle + Stripe)
`lovable.dev/changelog`: "Lovable payments lets you add subscriptions and one-time payments to any app using built-in Paddle or Stripe integrations."

**Impact:** Lovable now offers one-click payment scaffolding. This closes the "monetization gap" that previously required Stripe setup as a manual phase. vibesdk has no equivalent — users must implement payments from scratch.

**vibesdk gap:** Offer payment integration as a template choice at project creation (select "SaaS with payments" → pre-scaffold Stripe/Paddle setup in Phase 1).

### Desktop App (macOS, Windows coming)
Native macOS app with **local MCP server support**. Windows coming soon.

**Impact:** Local MCP server (vs. remote HTTP) = lower latency + richer context for IDE integration. Lovable is building toward native IDE embedding.

**vibesdk posture:** vibesdk's MCP server (POST /api/mcp, shipped S9) is remote. No native client needed given CF Workers architecture. No immediate action required.

### Claude Opus 4.7 Model Upgrade
Improved code quality + instruction following. Already confirmed in vibesdk config (vibesdk uses claude-3-7-sonnet for coding, Gemini 2.5 Pro for eval — no gap here).

### Group-Based Access Controls
Organize workspace members by team/role → manage project + folder access.

**Impact:** Enterprise gate feature. vibesdk's per-DO isolation is fundamentally stronger (BOLA immunity), but lacks the team UX layer.

### New Connectors (14 added April 24)
Google Workspace, Microsoft 365, BigQuery, Databricks, Snowflake, Asana, Ashby, HubSpot, Inngest, Resend, Fireflies, Gemini Enterprise, WordPress.

**Signal:** Lovable is building a data connector marketplace. Each connector = an additional LTV lock for enterprise accounts.
**vibesdk response:** Template-based connectors (scaffolded integration code) > proprietary connector marketplace. Maintain OSS advantage.

### Chat History Search (Semantic)
Retrieve past messages, run semantic queries across project conversation history.

**vibesdk gap:** vibesdk has per-session SQLite history but no cross-session semantic search. AgentMemoryClient (S8) provides recall, but no user-facing search UI.

## 2. Cursor: Canvases + Multi-Repo Cloud Agents (May 2026)

**Source:** [Cursor changelog](https://cursor.com/changelog)

### Cursor Canvases (new — May 13)
Agents create **interactive visual interfaces** (React-based) for data, PR reviews, eval analysis, and research — instead of dense text output. Built into the Agents Window.

**What it is:** When an agent completes analysis, instead of dumping markdown, it renders a React component (chart, table, form) directly in the chat panel. Users can interact with the output.

**vibesdk relevance:** HIGH novelty. vibesdk's phase quality signal (EvalGate) currently broadcasts raw scores via WebSocket. A "canvas" pattern would let vibesdk render an interactive quality dashboard inside the chat (phase score breakdown, file tree, quality trend). This is the next evolution of the PhaseQualityBadge component.

**Action (S11):** vibesdk equivalent: a "phase canvas" WebSocket message that renders a React component in the sidebar with eval scores, file diff preview, and action buttons (re-run eval, annotate phase).

### Multi-Repo Cloud Agents (May 13)
Cloud agents + automations now support **multi-repo environments**: configure one environment with all repos an agent needs, re-use across sessions.

**Impact:** Enterprise workflow — agent touches N repos per task. Not directly applicable to vibesdk's single-project-per-session model.

### Microsoft Teams Integration (@Cursor — May 11)
@Cursor in any Teams channel → delegates task to a cloud agent.

**vibesdk gap:** No team channel integration. Low priority (B2C + SMB focus). Track for enterprise pivot.

### Enterprise Admin Controls (May 4)
Model allow-lists, soft spend limits, per-user usage analytics.

**vibesdk equivalence:** vibesdk's CreditService + per-session cost_preview covers the billing side. Model allow-list not needed (single model per operation via AGENT_CONFIG).

### Bugbot Effort Customization
Default / High / Custom effort tiers for PR review.

**vibesdk equivalence:** EvalGate `reasoning_effort` in Gemini 2.5 Pro call. Already tunable via AGENT_CONFIG. No user-facing tier selector — add to S11 roadmap.

## 3. Replit Agent 4: Parallel Merge Resolution (launched March 2026, ongoing)

**Sources:** [Replit blog](https://blog.replit.com/introducing-agent-4-built-for-creativity), [Agent 4 launch](https://replit.com/agent4)

### Parallel Agent Architecture
Agent 4 dispatches parallel agents for auth, DB, backend, frontend simultaneously. Progress across tasks is clearly visible.

**Key claim:** "90% auto merge-conflict resolution" — parallel agents self-resolve merge conflicts using AI.

**ADR-007 validation:** Replit's 90% auto-merge claim validates the problem space. vibesdk's ADR-007 Option A (Phase Independence Constraint) avoids merge conflicts entirely by design — more conservative but **correct by construction**. ADR-007 Option B (LLM-Mediated Merge, S11) would be vibesdk's answer to the remaining 10% Replit still resolves manually.

**Counter-marketing angle:** "Replit resolves 90% of merge conflicts — vibesdk eliminates them. No merge = no risk of the 10%."

### Design Canvas (Infinite)
Generate design variants on an infinite canvas, tweak visually, apply best one to app.

**vibesdk equivalence:** Template selection at project creation is vibesdk's design input point. Canvas-driven iteration is a future UI feature (S12+).

### Multi-Artifact (web + mobile + decks + videos)
Same project produces multiple artifact types.

**vibesdk posture:** Single-artifact (web app) focus is a feature, not a gap — quality > quantity of output types.

## 4. Mobbin MCP: 621,500 Real UI Screens for AI Agents (May 13, 2026)

**Sources:** [BusinessWire](https://www.businesswire.com/news/home/20260511053592/en/Mobbin-Launches-MCP-Server-Giving-AI-Tools-621500-Real-App-Screens-to-Reference), [Morningstar](https://www.morningstar.com/news/business-wire/20260511053592)

### What It Is
Mobbin (design reference library) launched an MCP server providing access to **621,500+ app screens** and **142,200+ flows** from real shipped apps.

Supported categories: fintech, e-commerce, health, productivity, social, SaaS.

Integrates with: Claude, Cursor, Lovable.

**Setup:** Under 1 minute. Ask agent: "how do top apps handle paywall UX?" → Mobbin MCP pulls real references → agent builds from grounded patterns.

### Impact on vibesdk UI Quality
**This is the highest-impact finding in run020.**

Current vibesdk PhaseGeneration LLM prompt relies on text descriptions and Tailwind patterns for UI. Mobbin MCP would ground the UI generation in real-world patterns from 621,500 screens.

**Integration path (S10/S11):**
1. Add Mobbin MCP as an optional tool in the PhaseGeneration system prompt context
2. During phase planning, agent queries Mobbin for UI patterns matching the phase's domain (e.g., "auth flow screens" for an auth phase)
3. Returned screen references inform the `changes` field in `FileConceptSchema`

**Alternatively:** Simpler approach — pre-fetch Mobbin patterns for common vibesdk app categories at template selection time, embed as static UI context in the phase generation system prompt.

**Cost:** Mobbin paid plans only (no free tier for MCP). Pricing not confirmed.

**Action (S10 P1):** Evaluate Mobbin MCP integration. Even if not integrated directly, use Mobbin as editorial input to improve vibesdk's static UI guidance prompts.
