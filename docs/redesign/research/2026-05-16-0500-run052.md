```
# schema: plan|price|users|credits|collaboration

```

### Teams workspace architecture
```
# schema: role|permissions
Owner|Full admin; billing; invite/remove users; all project access
Admin|Invite/remove users; all project access
Editor|Manage and edit projects; NO invite/remove
Collaborator (Pro)|Can view/edit individual projects; uses project owner's credits
```

### vibesdk S15 Teams design implications
```
# schema: lovable_pattern|vibesdk_option_a_equivalent
20-user workspace with shared credits|DO-per-session with shared billing account linkage
Owner/admin/editor roles|WebSocket permission level on session WS connections
Real-time multiplayer editing|Multiple WS connections to same SimpleCodeGeneratorAgent DO (already supported via getWebSockets())
Shared credit pool|Billing service: workspace-level credit pool instead of per-user
Invite/remove users|User management endpoint + workspace membership table in D1
```

**Option A feasibility update:** Multiple WS connections to the same DO instance is already supported (`getWebSockets()` returns all active). Adding role-based permissions on top is the primary S15 work. This validates Option A as the correct MVP approach.
