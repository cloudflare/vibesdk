## 4. Architecture Decision: Email Templates (Production Readiness)

### Current state

ROADMAP Sprint 2 item: "Email templates: welcome, subscription-activated, cancelled, password-reset"
No email template implementation exists in the codebase.

### Assessment

Email delivery is NOT a Sprint 2 item for vibesdk — it's a "nice to have" for the first
cohort of paying users. Priority:
1. **Welcome email** — triggers on new user signup. Highest impact.
2. **Subscription activated** — confirms payment. Trust signal.
3. **Cancelled** — churn signal for analytics.
4. **Password reset** — required for security.

**Decision DEC-034-B:** Email templates are Sprint 2 P1 (not blocking for beta launch).
Resend.com is the recommended provider (Workers-compatible, $0 for first 3,000 emails/month).
Implementation: `worker/services/email/` — `EmailService.ts` + 4 template HTML files.
Estimated effort: 8h (templates + Resend API integration + wrangler secret).
