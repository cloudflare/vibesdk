## 1. Lovable Cloud — Previously Untracked Gap (Sep 2025 Launch)

**Source:** Lovable blog (fetched) — launched September 29, 2025.

**What it is:** Built-in managed backend for Lovable apps:
- User authentication + login
- Data persistence (database, row-level security)
- External service connections (safe, prompt-driven)
- Use cases: user profiles, messaging, file storage, media sharing
- Lovable AI (Google Gemini-powered): add AI features to apps without API key management

**Pricing:** Free tier covers $25/mo usage; usage-based billing above that.

**Gap analysis:**

```
# schema: dimension|lovable|vibesdk|verdict
Hosted auth|Built-in (prompt-driven)|Not hosted (generated code)|GAP
Hosted DB|Built-in Postgres-like|D1ProvisionService (generates code)|PARTIAL
Row-level security|Built-in|Generated code pattern|GAP
AI in generated app|Built-in (Gemini, no API key)|BYO API key (env var pattern)|GAP
Deployment|Lovable-hosted (no CF deploy needed)|User deploys to own CF account|TRADEOFF
Cost transparency|Usage-based (variable)|CF D1 + Workers (near-zero at scale)|AHEAD (cost)
Data sovereignty|Lovable-hosted infra|User's CF account (fully owned)|AHEAD (privacy)
```

**Classification:** S15 MEDIUM — not blocking Jul 1 launch (vibesdk targets developers who
WANT CF control); the D1ProvisionService (ADR-009) generates the database code and wrangler
setup. The gap is in "hosted convenience" for non-deployers. The Jul 1 audience (India indie
developer / tech-literate SMB) accepts CF deployment as part of the value, not a friction.

**Note:** This gap is 8 months old (Sep 2025). Previously untracked. Lovable Cloud is now
well-established in Lovable's product. Addressing in Jul 1 launch messaging: emphasize CF-native
as data sovereignty + cost advantage, not a deployment burden.
