### 2. phasic.ts — Parallel Dispatch Wiring (`313add1`)
- **File:** `worker/agents/core/behaviors/phasic.ts`
- Import: `executeParallelPhaseGroup`, `shouldEscalateToOptionB`, `type ParallelGroupMetrics`
- Added `parallelGroupMetrics` ephemeral field (not persisted, resets on DO restart)

  - Wraps `executeParallelPhaseGroup` using `this.executePhaseImplementation` as `runPhase`
  - State priority resolution: IDLE > REVIEWING > FINALIZING > PHASE_GENERATING
  - Logs escalation warning at >10% serialization rate (ADR-007 threshold)
- TypeScript: 0 errors

### 3. MCP Server — resources/list + resources/read (`5ee6d56`)
- **Files:** `worker/services/mcp/vibesdk-mcp-server.ts` + test
- MCP 2024-11-05 §resources support (Mastra v1.32 MCP Apps pattern)
- Resources capability declared in `initialize` response (server version 1.1.0)
- 3 URI templates (RFC 6570 Level 1):
  - `vibesdk://session/{sessionId}/status`
  - `vibesdk://session/{sessionId}/quality`
  - `vibesdk://session/{sessionId}/app`
- `parseResourceUri()` — validates protocol, authority, path segments
- `readResource()` — delegates to existing tool handlers (DRY, ownership preserved)
- `handleResourcesList()` + `handleResourcesRead()` + routing in `handleMcpRequest`
- Closes vibesdk_get_files gap (run019 P1); file-level resources deferred S11 (needs DO file-read RPC)
- **10 new unit tests** (resources/list ×3, resources/read ×7)
- TypeScript: 0 errors

### 4. Payments Scaffold — 'SaaS with Payments' use case (`f47542b`)
- **Files:** `worker/agents/schemas.ts`, `worker/agents/prompts.ts`, `worker/agents/planning/payments-scaffold.test.ts`
- Added `'SaaS with Payments'` to `TemplateSelectionSchema` `useCase` enum — LLM template selector now recognizes payment/subscription requests as this use case
- `SAAS_PAYMENTS_INSTRUCTIONS()` function — blueprint generation guidance for payments:
  - Pricing page (tier cards, monthly/annual toggle, feature matrix)
  - Stripe Checkout redirect flow — no PCI scope on frontend
  - Backend endpoints: `/api/billing/checkout` + `/api/billing/portal` + `/api/billing/webhook` via `fetch` (NOT Node.js stripe SDK — CF Workers incompatible)
  - `crypto.subtle` HMAC-SHA256 webhook signature verification pattern
  - D1 schema additions: `stripe_customer_id`, `subscription_status`, `plan_id`
  - Paddle as alternative (lighter global VAT handling)
  - Feature-gating pattern via `checkEntitlement()` helper
  - Explicit safety warning: `STRIPE_SECRET_KEY` must never reach the browser
- Wired into `getUsecaseSpecificInstructions` switch (new `case 'SaaS with Payments':`)
- **15 new unit tests** — schema acceptance (3), instruction content (12 assertions)
- Closes Lovable payments parity gap (run020 finding: Lovable ships Stripe/Paddle built-in April 24, 2026)
