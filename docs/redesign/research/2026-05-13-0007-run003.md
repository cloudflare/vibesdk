## TL;DR (5 bullets)
- Agent-memory market consolidated → Mem0 (91.6 LoCoMo), Zep/Graphiti (94.8 DMR), Letta (Context Repositories + git-versioned memory), Pinecone Nexus (Context Compiler) — vibesdk has **no memory layer at all**, biggest gap.
- Sandbox SOTA = sub-100ms cold-starts (Daytona 27-90ms, E2B Firecracker microVMs). vibesdk's Cloudflare Containers + isomorphic-git/SQLite is functional but lacks Firecracker-grade isolation tier & template marketplace.
- Cloudflare itself shipped Sandbox GA + Agent Memory + AI Search (hybrid retrieval) in Q2 2026 — vibesdk could swap custom stack for first-party primitives.
- Model-routing layer is table-stakes: OpenRouter (catalog), LiteLLM (self-host), Portkey (governance). vibesdk's `core.ts` uses bespoke BYOK plumbing — missing fallback chains, semantic cache, PII guards.
- Eval pipeline = zero in vibesdk. DeepEval (pytest-style, 50+ metrics) and RAGAS are the de-facto stack; competitors ship eval gates per phase.

## 1. Agent memory
### Mem0
- Latest: v1.0.1 (Nov 2025), Cassandra + FastEmbed local embeddings; OpenMemory Cloud shipped Jun 2025.
- Key new feature: 4-scope memory model (user/agent/run/app + org), multi-signal retrieval (semantic + BM25 + entity), MCP server callable from Claude Code.
- Bench: 91.6 LoCoMo (+20pt), 93.4 LongMemEval (+26pt).
- vibesdk relevance: drop-in long-term memory for `SimpleCodeGeneratorAgent`; would persist user style/prefs across sessions which today vanish per-DO.
- Source: [Mem0 State of AI Agent Memory 2026](https://mem0.ai/blog/state-of-ai-agent-memory-2026), [mem0 GitHub](https://github.com/mem0ai/mem0)

### Letta (formerly MemGPT)
- Latest: Conversations API (Jan 2026), Letta Code app (Apr 2026), Remote Environments (Mar 2026).
- Key feature: **Context Repositories** — programmatic context mgmt w/ git-based versioning. Three-tier memory: Core (RAM-like) / Recall (disk-cache) / Archival (tool-queried).
- vibesdk relevance: Letta Code is direct competitor — "memory-first coding agent". Their git-versioned context mirrors vibesdk's git approach but applies it to *agent state*, not source files. Worth studying.
- Source: [Letta's next phase](https://www.letta.com/blog/our-next-phase), [Letta Code blog](https://www.letta.com/blog/letta-code), [Rearchitecting Letta v1 agent loop](https://www.letta.com/blog/letta-v1-agent)

### Zep / Graphiti
- Bench: 94.8% DMR (beats MemGPT 93.4%). 20k+ GH stars, 25k weekly PyPI downloads.
- Key feature: **Bi-temporal knowledge graph** — every edge has (t_valid, t_invalid). Facts invalidated, not deleted → time-travel queries.
- vibesdk relevance: bi-temporal model fits vibesdk's phase-by-phase generation; could track "what blueprint claimed at phase N vs reality at phase N+3".
- Source: [Zep paper arXiv 2501.13956](https://arxiv.org/abs/2501.13956), [Graphiti GitHub](https://github.com/getzep/graphiti), [Neo4j Graphiti writeup](https://neo4j.com/blog/developer/graphiti-knowledge-graph-memory/)

### Pinecone
- Launched **Nexus** Knowledge Engine (Apr 2026): Context Compiler + Composable Retriever; >90 marketplace solutions at launch; Builder tier $20/mo; native full-text search in vector DB.
- Key feature: pre-compiles "derived artifacts" before agent needs them (vs query-time RAG).
- Compiled vector artifacts to accelerate agents (May 2026).
- vibesdk relevance: overkill for current vibesdk scope, but Nexus marketplace pattern (curated agent contexts) is a product-direction signal.
- Source: [Pinecone Nexus](https://www.pinecone.io/blog/knowledge-infrastructure-for-agents/), [Blocks & Files compiled artifacts](https://www.blocksandfiles.com/ai-ml/2026/05/05/pinecone-providing-compiled-vector-artifacts-to-accelerate-ai-agents/5219380)

## 2. RAG patterns
### LightRAG
- EMNLP 2025 paper; Mar 2026 added OpenSearch backend + setup wizard; Aug 2025 reranker support + doc-delete w/ KG regen.
- Integrates RAG-Anything for PDFs/Office/images/tables/formulas.
- vibesdk relevance: vibesdk has no RAG over template corpus / past projects → LightRAG could retrieve similar prior generations for cheaper context priming.
- Source: [LightRAG GitHub](https://github.com/hkuds/lightrag), [LightRAG site](https://lightrag.github.io/), [RAG-Anything PyPI](https://pypi.org/project/raganything/)

### ColPali
- Late-interaction multimodal embedding — treats page as 1024 patches, tensor-per-page. Now standard for visual-doc RAG.
- vibesdk relevance: low today (no doc-ingest); medium if vibesdk adds screenshot/wireframe ingest for spec→code.
- Source: [Infinity README discusses ColBERT-family rerankers incl. ColPali lineage](https://github.com/infiniflow/infinity)

### MinerU
- v3.0.0 (2026): service-based arch, AGPL-3.0, bundled PaddleOCR v3.4.0. Document-parsing engine RAG-Anything depends on.
- vibesdk relevance: niche unless vibesdk ingests user PDFs/specs.
- Source: [rag-api auto-selects MinerU/Docling/DeepSeek-OCR](https://github.com/BukeLy/rag-api)

### Infinity (Infiniflow)
- v0.7.0-dev3 (Feb 2026). 0.1ms query latency, 15k+ QPS on million-scale vectors; 1ms / 12k+ QPS full-text on 33M docs.
- Hybrid: dense + sparse + tensor + full-text + filtering; RRF/weighted/ColBERT rerankers.
- vibesdk relevance: serious alternative if vibesdk adds embeddings; D1 not designed for vector workloads.
- Source: [Infinity GitHub](https://github.com/infiniflow/infinity), [Infinity release notes](https://infiniflow.org/docs/release_notes)

## 3. Sandboxes (agent-execution)
### E2B
- Firecracker microVMs, per-exec dedicated Linux kernel. Largest template catalog. Native in OpenAI Agents SDK (Apr 2026).
- Best for "write→exec→observe" Python loops w/ shared kernel state.
- Source: [E2B site](https://e2b.dev/), [Help Net Security: OpenAI Agents SDK adds sandbox](https://www.helpnetsecurity.com/2026/04/16/openai-agents-sdk-harness-and-sandbox-update/)

### Daytona
- Pivoted to agent infra early 2025; **sub-90ms cold start**, optimized 27ms. $24M Series A Feb 2026.
- Native E2B+Daytona integrations in OpenAI Agents SDK.
- Source: [Northflank Daytona vs E2B 2026](https://northflank.com/blog/daytona-vs-e2b-ai-code-execution-sandboxes), [Superagent benchmark](https://www.superagent.sh/blog/ai-code-sandbox-benchmark-2026)

### GitHub Codespaces
- Devcontainer-as-code; **devcontainer-mcp** (2026) lets agents create/manage codespaces across local Docker + DevPod + Codespaces backends.
- Gitpod rebranded → **Ona** (Sep 2025), repositioned as "mission control for your team of SWE agents" — Codespaces alternative now agent-first.
- Source: [Codespaces features](https://github.com/features/codespaces/), [devcontainer-mcp](https://glama.ai/mcp/servers/aniongithub/devcontainer-mcp), [Bunnyshell alts 2026](https://www.bunnyshell.com/comparisons/github-codespaces-alternative/)

### vibesdk position
Cloudflare Sandbox GA (Q2 2026) gave vibesdk's host platform native equivalents: PTY over WS, R2 snapshots, active-CPU billing — but vibesdk built custom container/CLI tooling before this shipped. [Cloudflare Agents Week 2026](https://wal.sh/research/2026-q2-cloudflare-agents/)

## 4. Model routing
### OpenRouter
- One key → 200+ models; built-in routing, load-balance, fallbacks; pay per token.
- Best for: prototype/explore.

### LiteLLM
- OSS Python proxy, OpenAI-compatible for 100+ LLMs; YAML-defined fallback chains, round-robin/least-latency/cost-optimized; self-host (Docker + DB).
- Best for: full control, no external dep.

### Portkey
- Managed enterprise gateway: semantic caching, guardrails, PII filtering, governance.
- Best for: compliance-first orgs.

Source: [PkgPulse Portkey vs LiteLLM vs OpenRouter 2026](https://www.pkgpulse.com/guides/portkey-vs-litellm-vs-openrouter-llm-gateway-2026), [ToolHalla comparison](https://toolhalla.ai/blog/openrouter-vs-litellm-vs-portkey-2026)

vibesdk today: bespoke BYOK helper (`worker/api/controllers/modelConfig/byokHelper.ts`) + `inferutils/core.ts` — no fallback chains, no semantic cache, no governance.

## 5. Evaluation
### DeepEval
- 50+ metrics, **pytest-compatible** TDD framework for LLM/RAG/agents; bias/toxicity/tool-use metrics.
- Source: [DeepEval GitHub](https://github.com/confident-ai/deepeval), [DeepEval site](https://deepeval.com/)

### RAGAS
- Specialized RAG metrics: Faithfulness, ContextRecall, ContextPrecision. No bias/toxicity/tool-use.
- Source: [DeepEval vs RAGAS](https://deepeval.com/blog/deepeval-vs-ragas), [genai.qa comparison](https://genai.qa/blog/deepeval-vs-ragas/)

### TrustLLM
- BLOCKED — needs follow-up. Did not surface in 2026 evaluation-framework results; may be the academic benchmark (Sun et al. 2024) rather than an active framework. Not a peer of DeepEval/RAGAS in current tooling discourse.

MLflow now ships DeepEval, RAGAS, Phoenix as native judges. [MLflow third-party scorers](https://mlflow.org/blog/third-party-scorers)
