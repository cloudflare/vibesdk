## Key Technical Insight (MCP transport decision)

The `@modelcontextprotocol/sdk` v1.23.0 `MCPServerBase` has 7+ abstract methods
including `startStdio`, `startSSE`, `startHonoSSE`, `startHTTP`, `close`, `convertTools`, `getServerInfo`.
Implementing all of these for a CF Worker context would have taken 2+ days.

Alternative: implement the MCP JSON-RPC protocol (3 methods: `initialize`, `tools/list`, `tools/call`)
directly in ~250 lines. CF Workers compat guaranteed. Shipped in one iteration.

The `McpServer` high-level class from `@modelcontextprotocol/sdk/server/mcp.d.ts` also exists
as a concrete class with `.tool()` registration, but it requires a Transport implementation.
The stateless HTTP approach (no persistent connection) naturally fits CF Worker's request/response model.
