## What this means for vibesdk (architecture-only)

- **Cloudflare-native is NOT a unique moat at the marketing/CDN layer** — bolt.new, lovable.dev, replit.com all front on Cloudflare too. vibesdk's actual moat is **CF Workers + Durable Objects + D1 as the runtime backplane** (DO-per-chat-session, isomorphic-git in DO SQLite). No competitor probed runs the agent runtime on CF DO. Sharpen the messaging there, not on "we use Cloudflare."
- **Sandbox architecture is the real cost/perf battleground**: E2B Firecracker (200ms cold), StackBlitz WebContainers (in-browser Wasm, zero-server), Fly.io per-project (Blink), Devin's per-session Linux VM. vibesdk's custom container service should publish a cold-start number to compete on this axis.
- **CSP posture gap**: lovable.dev and manus.im ship strict CSPs (nonce + strict-dynamic). bolt.new ships none. Verify vibesdk's CSP — if absent, that is a quick-win parity move (`worker/config/security.ts` is in current diff — relevant).
