## UX shifts worth noting
- **Multi-repo cloud agent envs (Cursor May 13).** Cursor lets one config hold all repos an agent needs + reuse across sessions. Single-repo-per-session is now legacy. ([cursor.com/changelog](https://cursor.com/changelog))
- **Dockerfile-as-env-config w/ build secrets (Cursor May 13).** Replaces UI-driven env setup → reviewable, version-controlled, auditable. The "infra-as-code for AI agent envs" pattern is emerging.
- **Agent-led env setup w/ validation (Cursor May 13).** Agent provisions its own dev environment & validates before run starts. Pattern to copy in PhaseGeneration.
- **BYO-Browser (Manus May 12).** Lets user attach their own browser (presumably w/ saved auth/extensions) to Manus automation flows. Bypasses the cookie/auth problem.
- **Blink.new mirrors Lovable's tier shape exactly** ($25/$50/$200) → vibe-coding pricing has converged on this ladder. v0 ($30 team) & Cursor Pro ($20) sit just outside.

## Implications for vibesdk (delta-only)
- **Multi-repo workspaces are the new enterprise bar.** SimpleCodeGeneratorAgent assumes one project per DO. Add a workspace-spans-N-projects pattern OR a "session can attach additional repos" capability.
- **Env-as-Dockerfile w/ build secrets.** vibesdk container setup (`/container`) is template-driven — expose a user-editable Dockerfile + secrets injection from the User Secrets Store DO already built.
- **Agent-led env validation pre-run.** Add a PHASE_ENV_VALIDATE step before PHASE_GENERATING; fail fast on dep/secret/auth issues.
- **BYO-Browser hook (Manus pattern).** vibesdk preview iframe could accept a user-managed browser session for authenticated previews — niche but novel.
