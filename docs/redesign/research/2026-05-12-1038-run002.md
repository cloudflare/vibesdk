## Implications for vibesdk
- **Model-router UX is mandatory.** v0 exposes Mini/Pro/Max as plan-tier selectors; Cursor exposes Composer 2 vs frontier. vibesdk runs Opus 4.7 + others — surface a user-facing model selector w/ cost-per-task preview.
- **Sub-agent / parallel execution is the new bar.** Replit Agent 3, Cursor 3, Devin all spawn sub-agents. vibesdk's SimpleCodeGeneratorAgent is single-threaded per DO — design a sub-agent spawn pattern (separate DO instances?) before this becomes a checkbox loss.
- **Effort-based pricing beats flat credits.** Replit + Devin both shifted; users hate predictable simple-task overcharges. vibesdk's pricing should price by actual phase complexity not per-message.
- **Post-deploy ops loop is wide-open differentiator.** Only Replit App Monitoring closes the build→deploy→debug-from-prod-logs loop. vibesdk runs on Cloudflare → tap Logpush + Analytics Engine to give agent direct read on user's live app logs. Big moat.
- **Integration breadth = sticky.** Manus shipped 7+ connectors in 60 days (Slack, Meta Ads, IG, GWS). vibesdk has Cloudflare-native — add Supabase, Stripe, Resend, GitHub-as-deploy-target as first-class connectors w/ secrets store already built.
