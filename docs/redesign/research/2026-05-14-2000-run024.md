## Lovable — May 2026 Feature Launches

### Lovable Cloud (May 7)
Built-in backend abstraction layer. Users tell Lovable in natural language what they want; Lovable provisions storage, auth, and external service connections transparently.
- Gemini 3 Flash as default model (switchable per-prompt)
- Free promotional tier: $25 Cloud + $1 AI credits/month through May end (even free users)
- iOS + Android mobile app: start on desktop, continue on mobile

**vibesdk gap (HIGH):** vibesdk's D1 + Durable Objects architecture is technically equivalent — but it is NOT abstracted as a "built-in backend" product feature. Users must understand D1/DO to benefit. Lovable Cloud hides the infrastructure entirely. The gap is UX, not technical.

### Wiz Security Integration (May 7)
Native security scanning wired into every Lovable project's Security view. Wiz findings surface in the Security tab and flow back into Wiz Code and Build scans. Essentially: Lovable now runs a pentest on every deploy.

**vibesdk gap (MEDIUM):** vibesdk has architectural BOLA immunity (per-DO isolation) as a passive guarantee. Lovable now has active scanning as a visible feature. Users see "no vulnerabilities found" as a UI signal. vibesdk's advantage is stronger (architectural, not scan-based) but less visible.

**Action:** Surface the DO isolation guarantee as a UI signal — "Architecture-level isolation: zero shared state between projects." Counter-markets Wiz integration with depth, not breadth.

### Discoverability / SEO (May 13)
New SEO and AI search optimization for built apps. "Building is just the beginning" — Lovable now helps apps get found after launch.

**vibesdk gap (LOW):** Post-launch discoverability is out of scope for vibesdk's core loop. Deferred S12+.

### Agentic Development Standards (May 13)
Lovable co-authored governance standards with AIUC (AI User Community). Positioning play — Lovable as the "responsible" agentic platform.

**vibesdk signal (LOW/WATCH):** If AIUC standards become industry references, vibesdk needs to align. Monitor.

### Build in Parallel / `/multitask` (May 7, v3.3)
Simultaneous async subagent task execution for PRs. "/multitask" slash command. Redesigned PR interface.

**vibesdk gap (P0 — already wired, needs to ship):** ADR-007 Option A (disjoint-file parallel phases) + TeamLeadCoordinator are already implemented behind `multiAgentEnabled` flag. ## Google Stitch — DESIGN.md Protocol (Emerging Standard)

Stitch introduced **DESIGN.md** — an agent-friendly markdown file encoding design rules (colors, typography, component conventions). Export/import between Stitch and coding tools.

**Why this matters for vibesdk (HIGH):**
If Stitch generates DESIGN.md and users expect their coding tool to READ it, vibesdk needs to consume DESIGN.md from the project root and inject it into blueprint context. This is cheap (1 file read → prepend to system prompt), but if missed, vibesdk generates apps that violate the user's design system.

**Action (S11, effort 2h):** Add DESIGN.md detection to FileManager init path. If present, prepend to Planner system prompt under "### Design Rules". No format spec exists yet — treat as freeform markdown injection.

## NxCode — Free Tier Deep Profile

**Positioning:** "Full-stack for FREE" — non-expiring credits, code ownership, deploy anywhere.

**Target:** Non-technical founders (5,000+ claimed). Exact same audience as vibesdk.

**Free tier capabilities:**
- Full-stack generation: React/Next.js/Node.js/Python
- Architecture planning (schema, APIs, component hierarchy)
- Automated test generation + execution
- One-click deploy to Vercel/AWS

**vibesdk competitive response:**
```
# schema: dimension|NxCode|vibesdk|advantage
architecture|generic best-practices AI|DO-per-session, phase isolation|vibesdk
runtime|Vercel/AWS (shared infra)|CF Workers + DO (isolated)|vibesdk
parallel_phases|unknown|100% conflict-free ADR-007|vibesdk
BOLA_security|unknown|architectural isolation|vibesdk
credits|non-expiring|not yet defined|NxCode
code_ownership|full export|full ownership (isomorphic-git)|tied
pricing|free forever|freemium (TBD)|NxCode (for now)
```

**Key insight:** NxCode's moat is *not technical* — it's the "free forever" positioning for non-tech founders. vibesdk's moat is technical (DO isolation, conflict-free parallel phases, BOLA immunity). These are different purchase motivations. Counter-strategy: emphasize quality + reliability for founders who have already tried free tools and had a bad experience.
