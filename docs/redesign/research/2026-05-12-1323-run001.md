## Competitor Snapshots

### 1. emergent.sh
- **Pricing (verified):** Free $0/10cr · Standard $20/100cr · Pro $200/750cr · Team $300/1250cr · Enterprise custom. Annual saves ~17%.
- **Architecture:** GCP Cloud Run + App Engine + Supabase + Pusher + CF CDN (prior intel). Outputs React/Next.js/Python codebases (fully editable). Universal API Key brokers Stripe/Shopify/etc.
- **Recent (Apr 2026):** Launched **Wingman** — messaging-first autonomous agent (OpenClaw-class background task runner). E-2 Agent on Claude 3.5 Sonnet. Series A $70M @ $300M val from SoftBank/Khosla (Jan 2026).
- **BYO:** Custom domain + bring-your-own database verified on enterprise tier. SSO/SOC2/ISO27001 on Team+.
- **Weakness:** $200/mo Pro is the only tier w/ 1M context + system-prompt edit. 10 free credits is punitively low → conversion friction. No published per-credit cost in $.

### 2. manus.im
- **Status:** Meta $2B acquisition BLOCKED by China antitrust 2026-04-27; product running, ownership unresolved.
- **Architecture:** Multi-agent w/ central **Planner Agent** decomposing prompts → sub-tasks. Planner reasons about tool+params BEFORE each function call, injects reasoning into main agent context. **29 tools**. Core model **Claude 3.5 Sonnet v1** (only model that handled extended action→observation loops in their testing) + fine-tuned Qwen. Runs on **E2B Firecracker microVMs** (multi-hour persistent sessions).
- **Pricing:** Free 300 daily + 1000 starter credits, 1 concurrent task. Pro $20–$200/mo. Team $20/seat (2 seat min). Purchased credits never expire.
- **Weakness:** Credits don't roll over on free/pro. Heavy dependency on Claude — pricing pressure if Anthropic ToS changes. Tool selection accuracy is the bottleneck (27-tool abstraction).

### 3. bolt.new (StackBlitz)
- **Architecture:** **WebContainer** (browser-native Node, npm, terminal in IndexedDB). `/api/chat` endpoint → server context optimization → SSE stream → `EnhancedStreamingMessageParser` → `ActionRunner` executes in WebContainer. Clean parse/execute separation.
- **Recent:** 40% build perf improvement YoY (Jan 2026). **Secret Masking** layer added 2026 (API keys never leak to LLM training/preview URLs).
- **Deploy:** Defaults to Vite, in-browser HMR.
- **Weakness:** Browser-bound = no real OS-level processes, no GPU, RAM-bound to user device. Cannot do native compilation/heavy backend.

## Agentic Memory State of Art (Apr-May 2026)

**LOCOMO benchmark (Mem0 paper, ECAI 2025) — verified numbers:**

| System | Median latency | p95 | Tokens/conv | Accuracy |
|---|---|---|---|---|
| Full-context | 9.87s | 17.12s | ~26,000 | baseline |
| Mem0 (vector) | 0.71s | 1.44s | ~1,764 | 66.9% |
| Mem0g (graph) | 1.09s | 2.59s | ~1,800 | 68.4% |
| RAG baseline | 0.70s | — | — | 61.0% |
| OpenAI Memory | — | — | — | 52.9% |
| Letta (MemGPT successor) | — | — | — | ~83.2% |
| SuperLocalMemory Mode C | — | — | — | 87.7% |

**Architectural patterns ranked for vibesdk's per-session Durable Object + SQLite model:**

1. **Mem0 vector-only** — best fit. Sub-second p95 fits DO request cycles. ~1.8k tokens/conv keeps Workers CPU+egress costs low. 19 vector backends. SQLite-via-Durable-Object SQLite already in our `worker/agents/git/fs-adapter.ts` pattern → we can write a custom backend. **Downside:** Mem0 doesn't ship SQLite directly; we'd write the adapter.
2. **Letta three-tier (core/archival/recall)** — closest analogue to our `CodeGenState` model (project identity = core, generatedFilesMap = recall, git history = archival). Highest accuracy among OSS. Heavier to host.
3. **Zep** — handles summarization+entity-extraction async. 600k tokens/conv overhead = wrong for our per-session pattern. **Reject.**
4. **Graph memory (Mem0g, Cognee)** — multi-turn relationship reasoning. Overkill for single-app-session scope. Defer until cross-project memory becomes a product requirement.

**Recommendation for vibesdk:** Start with **Mem0 vector pattern, SQLite-backed in DO**. Mirror the `generatedFilesMap` index + add a conversation-summary vector table. Total added storage per DO: <2MB.

## 5 Highest-Leverage Insights for VibeSDK

1. **Replace custom sandbox w/ e2b.dev — $0.05/hr/1vCPU.** Manus already runs this stack. Free 100 credits to validate. Code change: rip out `/container/` tooling, swap `worker/services/sandbox/` to e2b SDK. Saves ~6 months of containerd maintenance. (BLOCKED — needs cost simulation against current Cloudflare container compute spend.)
2. **Adopt AG-UI Protocol for frontend↔agent comms.** Replaces bespoke WebSocket types in `worker/api/websocketTypes.ts`. Get Google/MS/Amazon-validated event schema for free. Marketing angle: "vibesdk speaks the open agent protocol."
3. **Differentiation gap = Cloudflare edge + Durable Objects.** Every competitor runs centralized clouds (GCP/AWS/Vercel). None offer per-session DO isolation + global edge. **Marketing line:** "your code-gen agent runs in the same edge region as your future users."
4. **Pricing wedge: undercut $200 emergent Pro / $100 Replit Pro at $49–79.** Their margins on credits suggest 60–80% gross margin on compute. We have Workers cost advantage. **Code change:** add a "Power" tier between current free and team in `worker/api/controllers/billing.ts` — verify migration paths in `worker/database/services/`.
5. **Ship Mem0-style memory layer in DO SQLite.** Adds long-session continuity that Bolt/v0 lack (browser-bound) and that Cursor doesn't have web parity for. Implementation: new `worker/agents/memory/` module, vector table in DO SQLite, hook into `UserConversationProcessor.ts`.
