## 5 highest-leverage architecture insights for vibesdk (ranked by ROI)

1. **Adopt token-metered billing UX (v0 + Replit signal).** v0 moved to in/out token credits; Replit moved to effort-based per-checkpoint. Both signal users want **predictable, sub-task granularity**. Vibesdk should expose per-phase cost previews. Touch: `worker/api/controllers/modelConfig/byokHelper.ts`, new `worker/services/billing/effortEstimator.ts`, `worker/api/websocketTypes.ts` (add `cost_preview` message).
2. **Bidirectional local↔cloud agent handoff (Cursor 3 signal).** Our `SimpleCodeGeneratorAgent` is Durable-Object bound — design parity by adding session-export/import contracts. Touch: `worker/agents/core/state.ts`, `worker/agents/core/websocket.ts`, new `worker/agents/core/handoff.ts`.
3. **Adopt AG-UI event protocol for our WebSocket layer.** Google/MS/Amazon/Oracle now back AG-UI; CopilotKit just raised $27M. Aligning our `STATE_SNAPSHOT`/`STATE_DELTA` semantics buys frontend interop + future SDK integrations. Touch: `worker/api/websocketTypes.ts`, `worker/agents/core/websocket.ts`, `src/routes/chat/utils/handle-websocket-message.ts`.
4. **DeepWiki-style effort-level ACU preview (Devin signal).** Show ACU-equivalent (= our credit) cost ranges *before* phase execution. Vibesdk currently runs phases without ex-ante cost transparency. Touch: `worker/agents/operations/PhaseGeneration.ts`, frontend `src/components/`.
5. **Messaging-first / mobile-async entry point (Emergent Wingman + Lovable mobile signal).** Two competitors shipped async/mobile interfaces in 30 days. Vibesdk's WebSocket model can be wrapped with a webhook receiver for WhatsApp/Telegram and a thin React Native shell. Touch: `worker/api/routes/index.ts` (add `/webhook/inbound`), `worker/agents/inferutils/core.ts` (decouple stream consumer).
