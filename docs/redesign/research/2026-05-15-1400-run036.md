## 1. Lovable 2.0 — CRITICAL COMPETITOR UPDATE (May 7, 2026)

### What shipped
- **Design view**: visual style editing, themes, image generation inside Lovable — live visual manipulation layer added on top of AI generation
- **Dev Mode**: direct in-browser code editing (no external IDE needed)
- **Workspaces / Teams**: Pro = personal workspace + unlimited collaborator invites; Teams = up to 20 members, role-based permissions (Owner/Admin/Editor), shared credit pools. Pricing: Pro from $25/mo, Teams from $30/mo.
- **Chat Mode Agent**: agentic reasoning without code edits — planning, debugging, file/log inspection. Multiple steps, no diffs emitted.
- **Security Scan**: vulnerability detection on publish (requires Supabase connection)
- **Wiz security integration** (May 13, 2026): Wiz scanning runs natively in Lovable Security view — surfaces vulnerabilities, secrets, misconfigs. Google Cloud partnership.
- **Custom domains**: purchasing + connecting built-in (10k+ domains already connected)
- **iOS/Android mobile app**: Lovable as a native app (meta-feature for their UX)
- **MCP integrations**: Notion, Linear, Confluence, Jira, n8n (May 7, 2026)

### Gap assessment vs vibesdk


### Counter-narrative
Lovable Design view = visual style editing for non-devs. vibesdk DESIGN.md = architect-grade design specification for owner-operators who want their rules enforced across every generated file. Different value proposition. The DESIGN.md approach is reproducible and version-controllable; Design view changes are one-shot.

Lovable+Wiz = "scan-based" security. vibesdk DO isolation = "structural" security — each session has its own SQLite, no shared attack surface for BOLA. SECURITY.md is the asset here. Publish it as a trust landing page (DEC-036-B).

## 2. Replit Design Canvas — GA Confirmed (March 11, 2026)

Agent 4 shipped Design Canvas as GA on March 11, 2026. Available on ALL plans including free tier.

### Canvas capabilities
- Infinite board, all artifact types, live preview
- Multi-select, hover/active state editing, responsive overrides
- Variant generation ("Generate variants" on any element — compare in-context)
- Canvas edits immediately update live codebase — no handoff layer
- Parallel AI agents (design + code simultaneously)

### Impact on vibesdk
Replit Design Canvas is more advanced than Lovable Design view (infinite canvas, variant generation, all plan tiers). vibesdk's gap here is structural — no visual canvas layer. However: Replit runs on GCP-backed Nix containers, not CF Workers. vibesdk's DO-native architecture is irreplaceable for CF Workers-native deployment. Different deployment targets.

**Assessment:** Replit Design Canvas is a P2 gap (S14+). India-first owner-operators building CF Workers SaaS don't need infinite canvas — they need fast generation + strong quality gates + easy Razorpay billing. DESIGN.md is the spec-driven alternative.

## 3. v0.app (Vercel) — Platform API Threat

February 2026 major rebuild:
- Renamed to v0.app
- Git panel: branch-per-chat, PR creation against main
- VS Code-style in-product editor
- Sandbox runtime importing any GitHub repo + Vercel env vars
- Database: Snowflake, AWS integrations (enterprise reporting)
- **Platform API**: "Build your own AI app builder with the v0 Platform API" — Vercel opens v0 as an infrastructure layer. Enterprises can build vibesdk-like tools on v0.
- Pricing: switched to token-based (Feb 2026)
- 4M users reported

### Platform API assessment
The v0 Platform API is a **meta-threat**: it allows others to build vibesdk-like wrappers on top of v0. However:
- v0 is Vercel/GCP-hosted — not CF Workers native
- No India-first pricing, no Razorpay/UPI integration
- Platform API requires Vercel infrastructure costs (not India moat)
- Token-based pricing at 4M users → expensive at scale for Indian SMBs

**Assessment:** LOW threat to vibesdk's India-first CF-native segment. The v0 Platform API targets US enterprise developers who want to white-label v0. Monitor every cycle; no action this sprint.
