## 1. Anthropic Opus 4.7 Fast Mode — API Confirmed

**Source:** platform.claude.com/docs/en/build-with-claude/fast-mode (official Anthropic docs, live as of 2026-05-15)

**What:**
- Real Anthropic API feature: `speed: "fast"` body param + beta header `anthropic-beta: fast-mode-2026-02-01`
- Endpoint: `POST /v1/messages` (standard Anthropic API, not a separate endpoint)
- Supported models: **claude-opus-4-7**, claude-opus-4-6 (note: 4-6 NOT in our current claudeDirect.ts — only 4-7 matters)
- Up to **2.5× higher output tokens per second** (OTPS improvement, NOT TTFT)
- Pricing: **6× standard Opus rates** → $30/MTok input, $150/MTok output
- Access: **waitlist-gated beta** (join at claude.com/fast-mode)
- Rate limits: separate from standard Opus — `anthropic-fast-input-tokens-limit` headers
- Response `usage.speed` field confirms which speed was served (`"fast"` or `"standard"`)
- Fallback pattern: catch `RateLimitError` → retry without `speed: "fast"` (different prompt cache bucket — cache miss on fallback)
- Incompatible with: Batch API, Priority Tier, Claude Platform on AWS

**vibesdk impact:**
- **HIGH LATENCY WIN if access granted.** PhaseImplementation is our UX latency bottleneck. BUT PhaseImplementation uses `core.ts` (OpenAI SDK routed through AI Gateway) not `claudeDirect.ts`. Fast mode is a NATIVE Anthropic API beta — not available via the OpenAI-compat layer.
- **Applicable today:** CriticAgent (uses `claudeDirect.ts` with `claude-opus-4-7`). CriticAgent is the sequential gatekeeper between phases. With fast mode: 2.5× faster critic = shorter inter-phase delay for Team+ sessions.
- **P2 — waitlist only:** Wire `speedMode: 'fast'` into CriticAgent immediately on waitlist approval. Code is ready: `claudeDirect.ts` now accepts `speedMode?: 'fast' | 'standard'` and sets `anthropic-beta: fast-mode-2026-02-01` header automatically.
- **Cost concern:** 6× pricing ($150/MTok output) makes fast mode too expensive for default use. Reserve for: CriticAgent (short output) and future paid tiers (Team+). NOT for PlannerAgent/CoderAgent (longer outputs).
- **NOT a solution for main code generation latency** — PhaseImplementation goes through `core.ts`/AI Gateway/OpenAI-compat, which doesn't support Anthropic's beta headers. Would need to migrate PhaseImplementation to `claudeDirect.ts` or add native Anthropic client path in `core.ts`.

## 2. CF AI Gateway Streaming Buffer — RFC Status Corrected

**Source:** github.com/cloudflare/agents/issues/1257 (RFC opened 2026-04-04, still open)

**What:**
- Status: **RFC — not yet implemented, not GA.** Agents Week blog described as roadmap direction.
- Proposed request ID header: **`X-AI-Gateway-Durable-Id`** (NOT `cf-aig-request-id` as assumed in ADR-008 draft)
- Resume API shape: **OPEN QUESTION** between two proposals:
  - Option A: `GET /gateway/buffer/{id}?offset={n}` — new endpoint
  - Option B: Reconnect with same request + `Resume: {id}` header — proxy extension
- Buffer TTL: **proposed 5 minutes** (configurable, matches WS reconnect window) — consistent with ADR-008 assumption
- Authentication mechanism: also open question in RFC

**vibesdk impact:**
- ADR-008 Phase 2 implementation is **BLOCKED** until RFC is merged and feature is GA
- ADR-008 updated to reflect correct proposed header name and confirmed-open endpoint question
- Re-check CF docs every cycle — RFC was recently active (comments from Cloudflare engineers, April-May 2026)
- The "GA from Agents Week" statement in ADR-008 original draft was **incorrect** — Agents Week described the streaming buffer as a roadmap item, not GA. Corrected.

## 3. OpenAI-Compat Layer and Fast Mode (gap analysis)

**Source:** claudeDirect.ts pattern analysis + Anthropic docs

**What:** vibesdk's main LLM path (`core.ts`) routes through the OpenAI SDK with CF AI Gateway as baseURL (OpenAI-compat format). Anthropic's fast mode requires:
- Native Anthropic API endpoint (not OpenAI-compat)
- Custom beta header not passable via OpenAI SDK
- Body field `speed: "fast"` not in OpenAI message format

**vibesdk impact:** To get fast mode on PhaseImplementation, one of:
- A. Add a parallel native Anthropic path in `core.ts` (complex — breaks provider abstraction)
- B. Route PhaseImplementation through `claudeDirect.ts` (simpler — migrate one AGENT_CONFIG action)  
- C. Wait for CF AI Gateway to support `anthropic-beta` header passthrough (no ETA)
- **Recommended:** Option B for a future sprint if fast mode waitlist access is granted and cost-benefit clears at Team+ pricing tier.
