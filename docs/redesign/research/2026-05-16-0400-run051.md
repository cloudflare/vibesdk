## 1. CF Unified Tracing — SHIPPED, GA, May 7, 2026

**Source:** developers.cloudflare.com/changelog/post/2026-05-07-automatic-tracing-across-do-and-worker-subrequests/

### What shipped (May 7 changelog)
```
# schema: capability|detail|vibesdk_relevance
Unified trace across DO + Worker subrequests|Single trace view; DO calls as nested child spans|CRITICAL — CodeGeneratorAgent DO + inference calls in one trace
Automatic — no code changes|Just wrangler config: observability.traces.enabled = true|ZERO code changes; pure config
OpenTelemetry-compliant spans|Timing, metadata, service bindings shown|Export to Axiom/Honeycomb/Grafana via OTLP
head_sampling_rate|0-1; 0.05 = 5% of requests traced|vibesdk: 0.1 (10%) — balanced cost/coverage
Billed from March 1, 2026|Included in Workers Paid plan usage|Workers Paid = vibesdk's current plan
Previously: disconnected traces|Separate trace per Worker/DO invocation|NOW: one trace ID spans the entire session processing chain
```

### vibesdk implementation (SHIPPED)
Added to `wrangler.jsonc`:
```jsonc
"observability": {
    "enabled": true,         // existing: logs + basic observability
    "head_sampling_rate": 1, // existing: 100% log sampling
    "traces": {
        "enabled": true,       // NEW: DO + Worker unified tracing
        "head_sampling_rate": 0.1  // 10% request sampling
    }
}
```

### What vibesdk gains
```
# schema: gain|detail
End-to-end session trace|HTTP request → codingAgent.ts → PhaseWorkflow → LLM call — one trace
DO subrequest spans|Each DO method call (onMessage, fetch, alarm) appears as child span
Service binding spans|Worker → DO RPC → sub-Worker all correlated
Zero code changes|No instrumentation needed; automatic from CF platform
OTLP export|Owner can configure Axiom/Honeycomb/Grafana via CF Dashboard → Workers → Logpush
```

### head_sampling_rate: 0.1 rationale
vibesdk has low concurrent traffic (S14/early S15 period). 10% sampling gives:
- ~1-5 traces/minute at expected early traffic
- Meaningful debugging capability for production issues
- ~10x cost reduction vs 100% sampling
- Owner can increase to 1.0 (100%) via `wrangler deploy` if incident occurs
