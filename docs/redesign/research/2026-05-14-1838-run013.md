## What this means for vibesdk
- **The CF-Workers tail-latency narrative is now public ammunition against us.** Theo Browne's "1 in 5 requests >10 s" claim [src 4] will be repeated by Vercel/v0 sales. vibesdk should preempt: publish our own benchmark for the SimpleCodeGeneratorAgent Durable Object hot path (p50/p95/p99 for streaming token, phase-gen, sandbox spin). Compare against Fluid w/ matched resource budget, not 128 MB-vs-2-vCPU.
- **Manus + Cognition both bet on microVM (Firecracker / custom hypervisor).** Our current sandbox in `worker/services/sandbox/factory.ts` is container-based. For long-running headless agents (>40 min sessions per Cognition's framing) we either (a) document a "we don't need 40-min state because DOs persist + git replays" message, or (b) plan a Firecracker-grade tier. Code touchpoint: `worker/services/sandbox/factory.ts` + `worker/agents/git/fs-adapter.ts` (the SQLite-FS adapter is the closest thing we have to a snapshot story — lean into it in messaging).
- **Replit Agent 4 = pipeline of sub-agents w/ specialized models** [src 5]. Our `worker/agents/operations/*` already mirrors this (PhaseGeneration / PhaseImplementation / UserConversationProcessor). We are architecturally aligned; the gap is **parallelism**. Today phases run serially. A `Promise.all` over independent phase slices would close the visible gap without changing the DO model. Touchpoint: `worker/agents/core/` state machine.
- **YC W26 "Terminal Use" ("Vercel for background agents") is the closest infra-layer threat.** Watch their launch — if they ship Firecracker-snapshot + per-agent VM as a primitive, that becomes the substrate every coding-agent startup will rent (E2B-style), commoditizing the layer we currently own end-to-end.
